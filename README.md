## Problem Statement
Ride-hailing platforms generate massive booking data daily. Without proper analysis, peak demand hours go undetected, cancellations increase, and revenue opportunities are missed. This project analyzes 5,000+ Ola ride records to uncover operational patterns and support business decisions.

## Dashboard Preview
![Dashboard Preview](dashboard.png)

## Key Insights
- Peak demand hours were **8–10 AM** and **6–9 PM** on weekdays
- Friday and Saturday had the highest ride volume — 28% above weekday average
- Cancellation rate was highest for bookings made more than 20 mins in advance
- Top 3 revenue-generating zones contributed 42% of total revenue
- Auto category had the highest volume; Prime Sedan had the highest average fare

## Tools Used
| Tool | Purpose |
|------|---------|
| Power BI | KPI dashboard, trend charts, map visuals |
| Excel | Data cleaning, Pivot Tables, source analysis |
| Power Query | Automated data transformation and refresh |

## Dataset
- 5,000+ ride booking records
- Fields: Booking ID, Date/Time, Pickup Zone, Drop Zone, Vehicle Category, Fare, Status (Completed / Cancelled)

## KPIs Tracked
- Total ride volume by hour, day, and week
- Cancellation rate by booking type and time slot
- Revenue by zone and vehicle category
- Average fare trends over time

## How to View
1. Download the `.pbix` file from this repo
2. Open in Power BI Desktop (free — download from microsoft.com)
3. Use the slicers to filter by date range, zone, and vehicle type
